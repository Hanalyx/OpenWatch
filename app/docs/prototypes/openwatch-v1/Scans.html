Auto-remediation Active

Kensa mutates the host, so auto-remediation is gated by policy. Choose per severity whether failed rules auto-fix, require approval, or stay off — and scope it by group. Only capturable handlers (full rollback) ever auto-apply.

Guardrails apply to every auto-fix. Non-capturable handlers (transactional: false) are never auto-applied; each fix still runs Capture → Apply → Validate → Commit with per-rule rollback.

Policy by severity

applies after each check
High
Highest-impact failures (root login, firewall off). Safest to review first.
Med
Config hardening with low blast radius (password policy, mount options).
Low
Cosmetic / advisory (banners, MOTD). Safe to auto-apply.
